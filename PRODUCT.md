# 产品介绍与使用文档

## 1. 产品概述

**2FA 在线工具 (ToTP Management System)** 是一个基于 Django 构建的企业级 TOTP（Time-based One-Time Password）密钥管理平台。它旨在解决团队协作中共享 2FA 密钥的痛点，提供安全、便捷的密钥存储、共享和审计功能。

核心价值：
- **集中管理**：统一存储个人和团队的 2FA 密钥。
- **安全共享**：通过团队空间安全地共享密钥，无需明文传输。
- **审计追踪**：详细记录密钥的查看、使用和导出行为。
- **便携访问**：支持移动端适配，随时随地获取验证码。

## 2. 核心功能特性

### 2.1 密钥管理
- **添加密钥**：支持手动输入密钥、识别二维码图片、粘贴 `otpauth://` URI。
- **分组管理**：支持自定义分组（如“工作”、“个人”），方便整理。
- **验证码生成**：实时生成 6 位动态验证码，支持点击复制和自动刷新。
- **回收站**：删除的密钥进入回收站，保留 30 天后自动清除，防止误删。

### 2.2 团队协作
- **团队空间**：创建多个团队，隔离不同项目的密钥。
- **成员管理**：邀请成员加入，设置不同角色（拥有者、管理员、成员）。
- **权限控制**：
  - **拥有者/管理员**：管理团队成员、添加/删除密钥、查看审计日志。
  - **成员**：仅查看和使用密钥验证码。

### 2.3 数据安全
- **加密存储**：密钥在数据库中加密存储（使用 Fernet 算法）。
- **二次验证 (Re-auth)**：执行敏感操作（如导出、查看密钥明文、删除）时需再次验证登录密码。
- **审计日志**：记录所有关键操作（创建、删除、导出、生成一次性链接等），支持导出审计记录。
- **一次性链接**：生成仅限单次访问或限时访问的密钥分享链接，用于临时协作。

### 2.4 导入与导出
- **批量导入**：支持从 1Password、Bitwarden、Authy 等主流工具导出的数据导入。
- **加密导出**：导出包含密钥明文的 JSON/CSV 文件，支持设置口令加密。
- **离线包**：生成单 HTML 文件的离线查看器，可在无网络环境下使用。

### 2.5 快速工具
- **免登录工具**：提供无需登录的临时验证码生成页面，数据仅在本地处理，不上传服务器。

## 3. 系统架构

系统采用经典的 Django MVT (Model-View-Template) 架构：

- **后端**：Python + Django
  - 使用 `django.contrib.auth` 进行用户认证。
  - 使用 `cryptography` 库进行数据加密。
- **前端**：Bootstrap 5 + 原生 JavaScript
  - 响应式设计，适配 PC 和移动端。
  - 使用 `QRCode.js` / `ZXing` 进行二维码识别。
- **数据库**：默认 SQLite（开发环境），生产环境推荐 PostgreSQL。
- **缓存**：支持 Redis 或本地内存缓存（用于限流和会话管理）。

## 4. 用户操作指南

### 4.1 注册与登录
1. 访问首页，点击右上角“注册”或“登录”。
2. 支持 Google One Tap 快捷登录（需配置）。

### 4.2 添加密钥
1. 进入“我的密钥”或“团队空间”。
2. 点击右上角“添加密钥”按钮。
3. 输入名称，粘贴密钥（Base32 字符串）或上传二维码图片。
4. 点击“保存”。

### 4.3 批量导入
1. 点击“批量导入”按钮。
2. 选择目标空间（个人或指定团队）。
3. 粘贴导出文本或上传文件。
4. 预览解析结果，确认无误后点击“确认导入”。

### 4.4 导出数据
1. 在列表页底部或设置菜单中选择“导出”。
2. 选择导出类型：
   - **加密导出**（推荐）：设置密码，生成加密文件。
   - **离线包**：生成 HTML 文件。
3. 输入登录密码进行二次确认。

### 4.5 团队管理
1. 导航栏选择“团队空间”。
2. 点击“创建团队”或进入已有团队。
3. 在“成员管理”标签页邀请新成员（通过邮箱）。

## 5. 常见问题解答 (FAQ)

**Q: 密钥安全吗？**
A: 密钥在数据库中经过加密存储。即使数据库泄露，没有 `SECRET_KEY` 和加密密钥也无法还原。建议生产环境务必妥善保管 `settings.py` 中的密钥配置。

**Q: 忘记密码怎么办？**
A: 如果配置了邮箱服务，可以通过登录页的“忘记密码”链接重置。

**Q: 离线包如何使用？**
A: 离线包是一个独立的 HTML 文件，下载后用浏览器打开即可。它包含加密的密钥数据，打开时需要输入导出时设置的口令。

**Q: 为什么部分操作需要二次确认密码？**
A: 为了防止会话劫持风险，涉及敏感数据（如导出密钥）的操作强制要求 5 分钟内的活跃认证。

## 6. 故障排除

- **验证码不正确**：请检查服务器时间是否同步，TOTP 极其依赖时间准确性。
- **无法发送邮件**：检查 `EMAIL_HOST` 等 SMTP 配置是否正确。
- **导入失败**：确认导入格式是否符合 `otpauth://` 标准或支持的 CSV 格式。
