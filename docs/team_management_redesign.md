# 团队管理模块系统性重构（现状盘点与设计规范）

## 目标与约束

### 目标

- 在不改变核心功能（团队/成员/邀请/审计/资产/分享链接）的前提下，重构信息架构与交互，使路径更短、层级更清晰、反馈更一致。
- 统一视觉语言与组件规范，提升一致性与可扩展性。
- 提升大团队/多团队场景性能：减少首屏 DOM、减少多余查询与无效刷新，让滚动与关键交互更顺。

### 约束

- 技术栈保持不变：Django 模板 + Bootstrap + 原生 JS/fetch。
- 安全流程不变：高风险操作继续走 reauth 与全局 confirm。

## 现状功能清单

### 团队

- 团队列表：搜索、分区（可管理/只读）、待处理邀请、新建团队、团队改名。
- 团队详情：概览、成员、资产、安全、审计入口。

### 成员与邀请

- 邀请（用户名/邮箱）：发送/更新/取消；受邀方接受/拒绝。
- 成员管理：角色切换（owner→admin/member）、移除成员、成员退出。
- 成员列表支持搜索与分页（片段加载 + 无刷新）。

### 安全与审计

- 撤销全部分享链接（高风险）：二段式摘要 → 全局 confirm → reauth（如需） → 执行撤销。
- 审计：筛选、分页、导出。
- Offcanvas 最近活动：分组/颜色标记，高风险置顶。

### 资产与归档

- 资产目录：搜索、创建/编辑/删除、负责人/关注人、密钥数量。
- 资产详情：已归属密钥 + 未归属密钥，支持批量归档/移除归档。
- 密钥在团队空间下可绑定资产（可选）。

## 用户流程（关键路径）

### 1. 进入团队并处理成员权限

1) 团队列表 → 进入团队  
2) 成员 Tab → 搜索定位成员 → 角色切换 / 移除 / 退出  
3) 成功后：页面局部刷新与计数一致

### 2. 风险治理：撤销分享链接

1) 团队操作侧边栏 → 高风险区点击“去撤销”  
2) Offcanvas 展示撤销摘要（有效链接数、最近创建者、最近创建/访问时间、访问状态）  
3) 点击“继续撤销”触发全局 confirm → 若需 reauth 进入确认页 → 返回后执行撤销  
4) 审计记录可追溯

## 新信息架构（IA）

### 两层结构

- 团队列表页：只聚焦“发现/选择团队 + 处理待办 + 创建团队”
- 团队工作台（Team Home）：固定团队上下文，把“成员/资产/安全/审计”作为同一团队内的主要任务入口

### 原则

- 始终明确“当前团队是谁”：页面标题/面包屑/角色徽章固定呈现。
- 将高频操作收敛为固定入口：顶部主按钮 + 右侧治理面板。
- 将低频/高风险操作放入治理面板：避免在多个位置重复。

## 交互规范（Best Practices）

### 加载与状态

- 列表/Tab 采用片段懒加载，首屏只渲染必要 DOM。
- 任何会影响列表一致性的操作优先以“刷新片段”来收敛状态，而不是拼接 delta。

### 反馈与错误

- 所有异步操作统一使用同一套提示（inline alert 优先，缺失则 toast）。
- 高风险操作：摘要 → confirm → reauth（如需）→ 成功提示 + 审计可追溯。

## 组件化 UI 规范（模板层）

### 页面结构

- PageHeader：标题 + 角色徽章 + 主按钮 + 次按钮
- ControlBar：搜索、过滤、统计 badge
- ContentTabs：主内容分区（overview/members/assets/security/audit）
- GovernancePanel：最近活动 + 风险提示 + 建议动作 + 快捷入口

### 设计令牌建议（CSS Variables）

- 圆角：8/12/16
- 阴影：轻/中/重（对应 panel/card/modal）
- 间距：4/8/12/16/24/32
- 字体：标题/正文/辅助文字（统一 line-height）
- 色彩：primary/secondary/info/warning/danger + subtle 背景，满足可读性对比度

## 无障碍与响应式

- 所有可点击元素保证可聚焦与可读 label（aria-label/aria-controls/aria-expanded）。
- Tab/Offcanvas 使用 bootstrap 语义组件，保持键盘可达。
- 移动端：优先单列，按钮允许换行，避免横向溢出。

## 性能与验证

### 性能策略

- 团队列表页避免渲染大量团队详情 DOM。
- 成员/安全/审计使用片段懒加载并支持无刷新翻页。
- 侧边栏面板采用服务端拼装 + 客户端加载，降低主页面模板复杂度。

### 测试策略

- 回归测试覆盖：成员分页/搜索、风险面板高风险置顶、二段式撤销摘要、无障碍 smoke。
- 全量测试通过后再合并样式迭代，避免“体验优化引入功能回归”。
